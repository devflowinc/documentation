# Files

Files are where most of the cards on Vault come from. These are stored on the server in an s3 bucket. On this page, we’ll dive into the different conversation endpoints you can use to manage files programmatically. We'll look at how to create, update, and delete files. {{ className: 'lead' }}

## The file model

The conversation model contains all the information about the files you upload. They can be linked to a specifc collection and you can access the file itself from each card in it. 

### Properties

<Properties>
  <Property name="id" type="string">
    Unique identifier for the conversation.
  </Property>
  <Property name="user_id" type="string">
    Unique identifier for uploader of the file.
  </Property>
  <Property name="file_name" type="string">
    The name of the file.
  </Property>
  <Property name="mime_type" type="string">
    The mime_tyoe of the file. (Only accepts .docx, .doc, .odt, .pdf)
  </Property>
  <Property name="private" type="boolean">
    Whether or not the file is visible to other users.
  </Property>
  <Property name="created_at" type="timestamp">
    Timestamp of when the file was created.
  </Property>
  <Property name="updated_at" type="timestamp">
    Timestamp of when the file was last updated.
  </Property>
  <Property name="size" type="integer">
    The size of the file in bytes.
  </Property>
  <Property name="base64url_content" type="string">
    The base64url encoded content of the file.
  </Property>
</Properties>

## Get a file {{ tag: 'GET', label: '/api/file/:file_id' }}

<Row>
  <Col>

    This endpoint allows you to retrieve a file by providing the file id. Refer to [the list](#the-file-model) at the top of this page to see which properties are included with file objects. (Must be authenticated if private)
  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/api/file/:file_id">

    ```bash {{ title: 'cURL' }}
      curl -L 'https://api.arguflow.com/api/file/9a981834-b12a-49cc-afec-4ce88a2b2456' \
        -H 'Cookie: ai-editor=your_cookie_value_here'
    ```

## Create a file {{ tag: 'POST', label: '/api/file' }}

<Row>
  <Col>

    This endpoint allows you to add a file to your collection. The file must be a .docx, .doc, .odt, or .pdf. The file will be stored in an s3 bucket and the file object will be stored in the database. (Must be authenticated)

    ### Required attributes

    <Properties>
      <Property name="base64_docx_file" type="string">
        The file you want to upload in base64url format.
      </Property>
      <Property name="file_name" type="string">
        The name of the file you want to upload.
      </Property>
      <Property name="file_mime_type" type="string">
        The mime type of the file you want to upload.
      </Property>
      <Property name="private" type="boolean">
        Whether or not the file is visible to other users.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="oc_file_path" type="string">
        The path to the opencaselist file the file is linked to.
      </Property>
    </Properties>
  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/api/file">

    ```bash {{ title: 'cURL' }}
      curl -L 'http://api.arguflow.com/api/file' \
        -H 'Content-Type: application/json' \
        -H 'Cookie: ai-editor=your_cookie_value_here' \
        --data '{
          "base64_docx_file": UEsDBBQABgAIAAAAIQDQWMtnkQEAAI8GAAATAAgCW0NvbnRlbnRfVHlwZXNdLnhtbCCiBAIooAACAAAAAAAAAAAAAAAAAAAAAAAAAAAA...,
          "file_name": "AI Affirmative - Northwestern 2018.docx",
          "file_mime_type": "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
          "private": false"
          }'
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
    {
      "id": "9a981834-b12a-49cc-afec-4ce88a2b2456",
      "user_id": "c9379ab0-389d-4204-8602-5ba1bf99d441",
      "file_name": "AI Affirmative - Northwestern 2018.docx",
      "mime_type": "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
      "private": false,
      "created_at": "2023-07-20T11:55:18.880672",
      "updated_at": "2023-07-20T11:55:18.880739",
      "size": 497097,
      "base64url_content": "UEsDBBQABgAIAAAAIQDQWMtnkQEAAI8GAAATAAgCW0NvbnRlbnRfVHlwZXNdLnhtbCCiBAIooAACAAAAAAAAAAAAAAAAAAAAAAAAAAAA..."
    }
    ```

  </Col>
</Row>

## Update a file {{ tag: 'PUT', label: '/api/file' }}

<Row>
  <Col>

    This endpoint allows you to update a file by providing the file id. (Must be authenticated)

    ### Required attributes

    <Properties>
      <Property name="file_id" type="string">
        The id of the file you want to edit.
      </Property>
      <Property name="private" type="boolean">
        Whether or not the file is visible to other users.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="PUT" label="/api/file">

    ```bash {{ title: 'cURL' }}
    curl -X PUT https://api.protocol.chat/api/file \
      -H 'Cookie: ai-editor=your_cookie_value_here' \
      -H 'Content-Type: application/json' \
      -d "{
        "file_id": "9a981834-b12a-49cc-afec-4ce88a2b2456",
        "private": true
      }"
    ```

    </CodeGroup>

  </Col>
</Row>
